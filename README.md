Данная программа выполнена в рамках практического задания учебного курса  
"Криптографические методы защиты информации" студентом Самариным Евгением.

## Описание

Программа реализует алгоритм асимметричного шифрования **RSA** (Rivest–Shamir–Adleman) на языке Go. Реализованы все основные этапы алгоритма:

- Генерация простых чисел с использованием вероятностного теста Миллера–Рабина
- Формирование ключевой пары (открытый и закрытый ключи)
- Шифрование сообщения
- Расшифрование шифротекста
- Сохранение и загрузка ключей в стандартных форматах PEM (PKCS#1)

## Возможности

- Генерация RSA-ключевой пары заданной разрядности (128, 256, 512 бит и более)
- Шифрование текстовых сообщений и файлов по открытому ключу
- Расшифрование шифротекста по закрытому ключу
- Сохранение ключей в форматах `RSA PUBLIC KEY` и `RSA PRIVATE KEY` (PEM/PKCS#1)
- Конвертация строк и файлов в/из шестнадцатеричного представления
- **Демонстрационный режим** — ввод параметров p, q, e и сообщения вручную с пошаговым отображением процесса шифрования и расшифрования RSA

## Использование

### Запуск

```bash
go run *.go
```

или после сборки:

```bash
go build -o rsa
./rsa
```

### Меню программы

После запуска отображается меню с опциями:

1. **Сгенерировать ключевую пару** — создаёт пару RSA-ключей указанной разрядности и сохраняет их в файлы `rsa-{bits}.pub` и `rsa-{bits}.pem`
2. **Зашифровать сообщение из командной строки** — шифрует короткое текстовое сообщение по открытому ключу
3. **Расшифровать сообщение из командной строки** — расшифровывает шифротекст (в hex-формате) по закрытому ключу
4. **Зашифровать файл** — шифрует содержимое файла по открытому ключу
5. **Расшифровать файл** — расшифровывает файл по закрытому ключу
6. **Демонстрационный режим** — шифрование малых чисел с отображением всех шагов RSA (ввод p, q, e вручную)
7. **Выход** — завершение программы

### Форматы ключей

- **Открытый ключ** — файл `rsa-{bits}.pub`, формат PEM `RSA PUBLIC KEY` (PKCS#1)
- **Закрытый ключ** — файл `rsa-{bits}.pem`, формат PEM `RSA PRIVATE KEY` (PKCS#1)

## Демонстрационный режим (пункт 6 меню)

Демонстрационный режим создан для наглядного отображения всех шагов алгоритма RSA при шифровании малых чисел. Позволяет вручную ввести параметры p, q, e и сообщение, после чего программа отображает полный процесс:

- Ввод простых чисел p и q
- Ввод открытой экспоненты e
- Вычисление модуля n = p * q
- Вычисление функции Эйлера φ(n) = (p-1) * (q-1)
- Проверка взаимной простоты e и φ(n)
- Вычисление закрытой экспоненты d = e^(-1) mod φ(n)
- Формирование ключевой пары
- Ввод сообщения (число или ASCII-символ)
- Шифрование: C = M^e mod n
- Расшифрование: M' = C^d mod n
- Проверка корректности расшифрования

Этот режим удобен для обучения и проверки правильности реализации алгоритма RSA на конкретных примерах.

## Алгоритм шифрования

Программа реализует классический RSA:

1. **Генерация ключей:**
   - Выбираются два случайных простых числа `p` и `q` заданной разрядности
   - Вычисляется модуль `n = p * q`
   - Вычисляется функция Эйлера `φ(n) = (p-1) * (q-1)`
   - Выбирается экспонента `e = 65537`, взаимно простая с `φ(n)`
   - Вычисляется закрытая экспонента `d = e^(-1) mod φ(n)`

2. **Шифрование:** `C = M^e mod n`

3. **Расшифрование:** `M = C^d mod n`

## Ограничения

Данная программа является учебным проектом и имеет следующие ограничения:

- **Размер шифруемых данных ограничен размером модуля RSA.** Шифрование выполняется по формуле `C = M^e mod n`, где `n` — модуль ключа. Сообщение (или содержимое файла) должно быть представлено числом `M < n`. Для ключа 512 бит это максимум 63 байта (64 байта минус 1 байт). Для ключа 1024 бит — 127 байт и т. д.
- **Отсутствие схемы заполнения (padding).** В данной реализации не используется схема PKCS#1 v1.5 или OAEP, которая в реальных реализациях RSA разделяет сообщение на блоки и добавляет криптографическое заполнение. Это означает, что программа может шифровать только одно сообщение, помещающееся в один блок RSA. Для практического применения необходимо использовать блочное шифрование с padding (например, PKCS#1 v1.5 или OAEP), что выходит за рамки учебного проекта.
- **Не оптимизировано по скорости и потреблению ресурсов.** Код не проходил оптимизацию и не предназначен для использования в промышленных системах.
